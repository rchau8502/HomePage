Notes

Rules I keep returning to

Make the hierarchy obvious

A good homepage should explain who you are, what you build, and where the user should click within seconds.

Use motion with restraint

A few meaningful animations carry more weight than constant movement that competes with the content.

Let the repo carry proof

When the code is public, the homepage gets credibility from the build history instead of relying on empty claims.
